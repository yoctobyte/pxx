# PXX v0.1.0-beta.1 — codename "<TBD>"

A from-scratch, self-hosting Pascal-dialect compiler (FPC-seeded) with its own RTL,
multiple code-generation targets, and a reproducible-build guarantee. First public
beta — `0.x` means the language, ABI, and CLI may still change between releases.

## Highlights

- **Self-hosting, byte-identical.** The compiler compiles itself to a fixed point;
  `make test` proves gen-N == gen-N+1 bit-for-bit.
- **Six code-generation targets** from one host: x86-64, i386, aarch64, arm32, and
  the ESP32 pair xtensa / riscv32 (emit-only).
- **Own RTL + PCL** (compiled from source, no host libc dependency for the compiler
  itself): strings, collections, math, sysutils, bitset, random, zlib, PNG, a GTK3
  widget set, an ANSI terminal/renderer, and a POSIX platform-abstraction layer.
- **ESP32 floating point + interrupts.** Single/Double/Real with soft-float kernels,
  xtensa hardware single FPU (`--xtensa-fpu`), `{$FASTDOUBLES}`, and raw `interrupt;`
  / `iram;` ISR codegen.
- **Reproducible builds.** Every target binary in `MANIFEST.sha256` reproduces
  bit-for-bit from any host (codegen is host-independent) — verify the whole release
  on one machine with `./selfcheck.sh`.

## Platform support

This release is built and **tested natively only on x86-64 (amd64) Linux**, where
the compiler self-hosts byte-identically and its emitted programs run natively.

All other targets are **validated under emulation, not on physical hardware**:

| Target               | How it's exercised                                | Status        |
|----------------------|---------------------------------------------------|---------------|
| x86-64 (amd64)       | native                                            | tested        |
| i386                 | kernel IA-32 (32-bit) compat layer; QEMU fallback | tested (emulated/compat) |
| aarch64, arm32       | QEMU user-mode                                    | experimental  |
| ESP32 xtensa/riscv32 | QEMU system (esp32s3 / esp32c3)                   | experimental  |

Reproducibility is broader than execution testing: **every** target binary in the
manifest reproduces bit-for-bit from an amd64 host, so the whole release is
verifiable on one machine. But only amd64 has been run on real silicon — treat
aarch64 / arm32 / ESP as beta until validated on hardware.

## Install

```sh
tar xzf pxx-v0.1.0-beta.1.tar.gz
cd pxx-v0.1.0-beta.1
./setup.sh                 # detect native arch, put `pxx` on PATH
pxx examples/primes/sieve.pas /tmp/sieve && /tmp/sieve
./selfcheck.sh             # self-fixedpoint + reproduce the release from this host
```

The compiler finds its RTL/PCL/builtin relative to its own binary (`ExeDir`), so the
unpacked tree works in place from anywhere — no environment variables required.

## Known gaps

- `examples/chess/chess.pas` does not compile — blocked on local typed constants
  (`feature-local-typed-constant`). Tracked in the release xfail registry.
- ESP32 `interrupt;` is implemented + structurally verified, but live-fire interrupt
  delivery is not yet exercisable from PXX (no on-target vector-table install path).
- xtensa `interrupt;` is Call0-only; the windowed-ABI ISR path is deferred.
- No macOS / Windows hosts (the compiler emits Linux ELF).

## Verifying this release

```sh
./selfcheck.sh
# check 1: self-fixedpoint (determinism on your CPU/kernel)
# check 2: reproduce every target binary vs MANIFEST.sha256 (bit compatibility)
```
A PASS means your machine independently reproduces the shipped binaries bit-for-bit.

## Acknowledgments

This project stands on a long lineage of people who made Pascal what it is.

- **Niklaus Wirth** (1934–2024), who designed Pascal in 1970 — and Modula-2 and
  Oberon after it. A teaching language built for clarity and structure that turned
  out to be powerful enough to write compilers, operating systems, and this. The
  whole point of PXX is an inheritance of his.
- **Blaise Pascal** (1623–1662), mathematician and builder of one of the first
  mechanical calculators — the name the language, and this compiler, carry forward.
- **Anders Hejlsberg**, who wrote the Turbo Pascal compiler and was the chief
  architect of Delphi — and later of C# and TypeScript. Turbo Pascal put a fast,
  affordable compiler in front of a generation of programmers; Delphi showed how far
  Object Pascal could go. That craftsmanship is part of this language's DNA.
- The **Borland / Turbo Pascal / Delphi** heritage — the conventions, the IDE-driven
  RAD model, and the Object Pascal dialect this project deliberately stays close to.
- The **Free Pascal Compiler (FPC)** team and community — FPC is the seed that
  bootstrapped PXX (the first generation was compiled by FPC before it became
  self-hosting), and its dialect compatibility, conventions, and decades of ecosystem
  work are the shoulders this stands on directly.

PXX uses FPC only as the **bootstrap compiler** and follows FPC-compatible
language/naming conventions. Its runtime library, PCL widget set, and backends are
**written from scratch** — no FPC RTL or LCL (nor any Borland/Delphi) source is
copied or ported. The gratitude is for the language, the ideas, and the foundation —
not for lifted code.

Thank you, all of you.
