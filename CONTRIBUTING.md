# Contributing

Contributions are welcome. Two rules keep the project's licensing clean:

1. **Sign your commits (DCO).** Every commit in a pull request must carry a
   `Signed-off-by:` line (`git commit -s`), certifying the
   [Developer Certificate of Origin](https://developercertificate.org/):
   you wrote the change or otherwise have the right to submit it under the
   license of the files it touches.
2. **Match the file's license.** Each file declares its license in an SPDX
   header (see [LICENSE.md](LICENSE.md)). New files take the license of
   their directory.

Do not submit code copied from other projects unless its license permits it
and you say so in the commit message.
